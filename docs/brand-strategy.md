# RunOwl — Brand Strategy

## Positioning

RunOwl is positioned as the **AI code review agent for engineering teams who ship fast and can't afford bugs in production**.

Not just a linter. Not just static analysis. A reasoning agent that reads your PR the way a senior engineer would — with context, curiosity, and a checklist.

## Voice & Tone

- **Direct**: No fluff. Lead with the finding, the severity, the fix.
- **Confident but not arrogant**: We catch real bugs. We don't cry wolf.
- **Technical**: We talk to engineers. Use precise language — "P0 critical", "OWASP A03", "race condition".
- **Dry wit**: A touch of personality in empty states, tooltips, and onboarding. Never at the expense of clarity.

## Core Messaging

### Primary tagline
> AI code review that finds what human reviewers miss.

### Secondary messages
- Catches bugs, security holes, and design issues before they reach production
- Reviews PRs the way a senior engineer would — recursively, with full repo context
- Free forever for open-source. Scales with your team.

## Brand Attributes

| Attribute | Description |
|---|---|
| Precision | Every finding is cited (file path + line number). No vague warnings. |
| Speed | Reviews complete in seconds, not minutes. |
| Depth | Goes beyond surface checks — follows imports, checks call sites, reasons about architecture. |
| Openness | Free tier is genuinely useful. Open-source core. |

## Competitive Positioning

| vs. | Differentiator |
|---|---|
| GitHub Copilot | RunOwl reviews the whole PR, not just autocompletes lines. Full OWASP + SOLID analysis. |
| CodeRabbit | Deeper reasoning via recursive RLM loop. Test generation + execution, not just review. |
| Manual review | Catches things humans miss at 2am. Consistent. Never tired. |
| SonarQube | No config. No rules files. Just point at a PR URL. |

## Visual Identity

- **Primary colour**: Deep teal / dark accent
- **Dark mode first**: Default theme is dark. Light mode available.
- **Typography**: Monospace for code, system sans-serif for UI
- **Icons**: lucide-svelte — consistent, clean, 24px stroke-based

## Audience Segmentation

### Primary: Engineering Team Leads (5–20 devs)
- Pain: PR review bottleneck, missed bugs in production
- Message: "Stop waiting for reviews. Start shipping with confidence."
- CTA: Start free → upgrade to Team when you hit the 10 PR limit

### Secondary: Mid-market CTOs / VPs of Engineering (20–100 devs)
- Pain: Inconsistent code quality, security incidents, compliance pressure
- Message: "OWASP coverage. SOLID analysis. Audit logs. One tool."
- CTA: Book a demo → Business or Enterprise

### Tertiary: Open-Source Maintainers
- Pain: Can't review every contribution thoroughly
- Message: "RunOwl reviews PRs so you can focus on direction."
- CTA: Free forever. No credit card.

## Content Strategy

- **Launch blog post**: Technical deep dive — how the recursive reasoning loop works
- **Product Hunt listing**: Feature-focused, lead with the testing engine differentiation
- **GitHub README**: Developer-first, CLI examples first, feature list second
- **Docs site**: Task-oriented. "How do I review a private repo?" not "Welcome to RunOwl."
